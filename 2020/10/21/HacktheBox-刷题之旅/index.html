hackthebox 获取邀请码：

查看网页源码发现一段js代码，带有invite提示，不过可读性很差，放到xor网站中美化一下代码，如下

function makeInviteCode() {
$.ajax({
type: “POST”,
dataType: “json”,
url: ‘/api/invite/how/to/generate’,
success: function (a) {
console.log(a)
},
error: function (a) {
console.log(a)
}
})
}

发现是个函数，在console中运行这个函数，可以获得一段加密后的密文，密文后面有说明是那种加密方式

提示是ROT13加密，把密文放到Rot13解密的网站上跑一下

解出来的明文提示要发送一个POST请求到 “/api/invite/generate”中
然后在控制台中输入 $.post(“api/invite/generate”, e=>{console.log(e)})

之后又可以获得一段密文，base64编码，解密后即为邀请码了

import requests

r = requests.session()

开启会话，具体作用未知，咕

out = r.get(url)
返回url的一个对象，有多种属性
===========================================
r.status_code ：HTTP请求的返回状态，200表示连接成功，404表示失败

r.text ：HTTP响应内容的字符串形式，即，url对应的页面内容根据，r.encoding显示网页内容

r.encoding ：从HTTP header中猜测的响应内容编码方式，如果header中不存在charset，则认为编码为ISO‐8859‐1

r.apparent_encoding： 从内容中分析出的响应内容编码方式（备选编码方式）

r.content ：HTTP响应内容的二进制形式

python re模块部分解释
out = re.search(“

+.*?

“, out.text)

search模式解释：search仅仅查找第一次匹配，也就是说一个字符串中包含多个模式的匹配，也只会返回第一个匹配的结果，如果要返回所有的结果，最简单的方法就是findall方法，也可以使用finditer方法

FreeLancer

观察html 发现有几个注释点都有<!– Portfolio 3 –>
可以试试id 这个参数是否存在sql注入

输入http://165.232.101.10:32338/portfolio.php?id=1 or true –
会有一段文字显示出来，如果不加 or true – 的话，只有一小段，判断存在数字型的sql注入

接下来尝试使用联合查询看看有几个字段
payload : http://165.232.101.10:32338/portfolio.php?id=1 order by 1..2..3
试到4的时候无数据显示，说明只有三个字段

知道有3个字段名后开始报出数据库名
payload：http://165.232.101.10:32338/portfolio.php?id=1 union select 1, 2, database() –

在回应数据的末尾出现数据库名 freelancer

由于数据库版本小于5.0，所以不存在 information_schema ，需要利用二分法等方法爆破表名，太麻烦就直接使用sqlmap跑

➜ hackthebox sqlmap -u http://165.232.101.10:32338/portfolio.php\?id\=1 -D freelancer –dump
-D 指定数据库名

跑下来发现freelancer 存在 safeadmin 和 portfolio 两张表

也可以使用 sqlmap -u http://165.232.101.10:32338/portfolio.php\?id\=1 –tables
可以直接跑出有哪几个数据库以及这几个数据库中各有几张表，数据库名和表名都清楚的知道

接下来尝试用john 破解密码，但是这狗日的工具不会用

好奇writeup 中是怎么发现administrat 目录的，搜了下发现有个dirBuster 可以扫描目录

此工具的使用：

  1. 要使用字典
  2. 勾选 URL fuzz
  3. 在最后一行的 / 后面 加 {dir}

最后点击start 开始爆破

还有一个owasp zap工具也可以扫描网站，不过没法把全部文件扫出来

dirmap工具也可以扫，也要依赖强大的字典
一般用法：
python3 dirmap.py -i http://46.101.16.203:31725/ -lcf

感觉本题的考点就是 发现sql注入， 爆破目录，熟练使用sqlmap工具

emmmm
本题不知道该怎么入手，看到登录注册框就想着手动注入一下，好像没什么用，想了会还是去找其他师傅的博客跟着学吧

首先对登录注册行动进行抓包，在注册成功后 进行登录的时候进行抓包，可以看到cookie的值非常长，采用的是jwt编码，
把这段jwt放到 jwtio 中会进行解码，可以获得headers， payload，根据博客里面的介绍，jwt是有个漏洞的，可以通过改
头部的算法的方式，将RSA加密算法修改为HMAC，这样的结果就是服务端会直接使用HS256，并且把传过去的公钥（RSA
的公钥）看作是当前算法（HMAC）的密钥来进行验签。

这样就可以将注入语句放在username中，之后再用RSP的公钥对 payload 进行HMAC加密，发送到服务端后注入语句就
会发生作用。

一开始的注入语句 “root’ union select 1, 2, database() –” 试了下，发现报错，表示sqlite没有这个函数

最后就是读取字段 top_secret_flaag 的值get flag了

getToken：作用是对 passwordHash + PublicKey 进行 sha256加密，用python写也就是 sha256(passwordHash + PublicKey)

要得出token需要 passwordHash 与 PublicKey，那么再看看这两个东西是怎么计算来的

passwordHash: 在getPasswordHash函数中对 password + PASSWORD_HASH_SALT 进行sha256加密得来
PublicKey: 在getPublicKey函数中对ClientUid + passwordHash 进行sha256加密得来

整合一下得到 token = sha256(sha256(password + PASSWORD_HASH_SALT) + PublicKey)

little Tommy

程序保护：

32位程序，除了NX其他保护全关

程序分析：
在delete功能有一个明显的uaf漏洞，通过nc尝试发现无法进行double free， 推测目标机器的libc版本小于2.27（好像现在libc2.27也做了检查无法简单的double free了…)

继续审查程序，发现存在system函数，只要满足*((_DWORD *)main_account + 16) == 1801680230 就可以夺取flag，由于 create功能 限制了可输入字符串的长度，所以无法正常满足 *((_DWORD *)main_account + 16) == 1801680230，但是可以通过strdup函数来控制 malloc(0x48) 这个chunk的data区域，因为strdup函数的原理是根据要复制的字符串的长度来申请chunk， 之后再调用strcpy函数复制到chunk中。所以只要把create 的chunk给free掉，再控制&s 的字符串长度为0x48，这样调用strdup函数中，输入的数据就可以控制 *((_DWORD *)main_account + 16)
为 “fuck”(0x6B637566) 了

Dream Diary: Chapter 2

附件不给libc我是没想到的，它不给libc我也不想去测目标机器上的libc版本是多少，直接按照自己本地的libc来做
本地环境为 Ubuntu16.04 ，libc-2.23

程序保护：
开了CANARY和NX，其他保护关闭

漏洞点：
在sub_400B12 函数中存在null by one 漏洞

解题思路：

  1. 泄露libc地址和堆地址
    本题是个菜单题，各个功能都不缺，所以libc地址可以直接free一个chunk到unsorted bin中再取出来，调用dump函数就可以泄露

至于堆地址，先free一个size不为0x21的chunk， 之后再创建一个size 为0x21的chunk，free掉，这样fast bin中的大小为0x20的链表中存在三个chunk，只要再创建一个size为0x21的chunk，就可以控制上面那个size为0x21的chunk，bk指针还残留着heap地址，填充fd，再调用dump函数就可以泄露堆地址了

  1. 利用House of Einherjar && fast bin attack 来getshell
    House of Einherjar：
    伪造一个fake chunk，控制fd,bk指针为 &(fake chunk)，绕过unlink检测

之后利用null by one 漏洞来踩要触发 House of Einherjar 的那个chunk的标志位，再free掉这个chunk，就会触发House of Einherjar，造成chunk overlap

之后就简单了，利用chunk overlap 控制fast bin的fd指针指向malloc_hook - 0x23，之后覆盖malloc_hook为one_gadget就可以getshell了
